Vulnerability catalogue

SQLi

SQL Injection in login & signup

Queries in auth_service.py are built with string concatenation. A username of ' OR '1'='1 bypasses authentication.

XSS

Stored XSS via dashboard username

The {{username}} placeholder is replaced with str.replace in routes/auth.py with no HTML escaping, so a malicious username executes on every visit.

XSS

Reflected XSS via /search

The q query parameter is interpolated into both the SQL LIKE clause and the HTML response body without escaping.

Session

Session hijacking via hardcoded secret

SessionMiddleware is initialised with the literal super-secret-key-12345 in main.py. Anyone with the secret can forge cookies.

Crypto

Weak password storage (MD5, no salt)

Passwords are hashed with plain MD5 in core/security.py. No salt, no work factor — a rainbow table defeats every account.

Brute

No rate limiting on any endpoint

No rate-limit middleware, no failed-login backoff. An attacker can brute-force credentials as fast as the network allows.

Exposed

Unauthenticated database download

GET /download/db returns the SQLite file with no session check. Pull the entire user table (including MD5 hashes) in one request.

CSRF

No CSRF tokens on any form

Login and signup forms have no hidden CSRF token. A cross-site form post can drive the victim's authenticated session.
